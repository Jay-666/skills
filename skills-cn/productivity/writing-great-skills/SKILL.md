skill 的存在是为了从随机系统中驾驭确定性。**可预测性**，即 agent 每次运行采用相同的*过程*而非产出相同输出，是根本美德；下方每个杠杆都服务于它。

**加粗术语**在 [`GLOSSARY.md`](GLOSSARY.md) 中定义；在那里查看完整含义。

## 调用

两种选择，分别付出不同成本：

- **模型调用**的 skill 保留**描述**，因此 agent 可自主触发它，其他 skills 也可访问它（你仍可输入其名称）。它增加**上下文负荷**，因为描述每轮都在窗口中。机制：省略 `disable-model-invocation`，并编写包含丰富触发措辞的面向模型描述（“当用户希望……、提到……时使用”）。
- **用户调用**的 skill 将描述从 agent 的可及范围移除，只有你输入其名称才能调用，其他 skill 也不能调用。它没有上下文负荷，但消耗**认知负荷**，因为*你*是必须记住它存在的索引。机制：设置 `disable-model-invocation: true`；`description` 成为面向人的一行摘要，移除触发列表。

仅当 agent 必须自行使用此 skill，或其他 skill 必须使用时，才选择模型调用。若它仅会由人工触发，则设为用户调用，不承担上下文负荷。

当用户调用的 skills 多到超出可记忆范围时，可用**路由 skill**解决累积的认知负荷：一个用户调用的 skill，列出其他 skills 以及何时使用各自。

## 编写描述

模型调用的**描述**做两项工作：说明 skill 是什么，并列出应触发它的**分支**。每个词都增加**上下文负荷**，因此描述比正文更需要裁剪：

- **将 skill 的引导词置于前面**，描述在此完成调用工作。
- **每个分支一个触发条件。** 为同一分支重命名的同义词是**重复**，例如“使用 TDD 构建功能……要求测试优先开发”是同一分支写了两次。合并它们，仅保留真正不同的分支。
- **移除正文已有的身份信息。** 描述只保留触发条件，加上所有“当另一 skill 需要……”的访问条款。

## 信息层级

skill 由两类可自由混合的内容构成：**步骤**和**参考**。skill 可以全是步骤、全是参考或两者兼有。核心决策是使用何者，以及每项在**信息层级**中的位置。信息层级是一把按 agent 需要材料的即时程度排序的梯子：

1. **skill 内步骤**：`SKILL.md` 中的有序动作，是主要层级，表示 agent 按顺序做什么。每个步骤以**完成标准**结束，即告知 agent 工作完成的条件。使其*可检查*（agent 能分辨完成与未完成吗？），并在重要时*穷尽*（“已覆盖每个修改模型”，而非“生成变更列表”），模糊标准会引发**过早完成**。
2. **skill 内参考**：`SKILL.md` 中按需查阅的定义、规则或事实。通常是合理的平级集合（审查的每条规则位于同一层），这是一种良好安排而非异味。*此 skill 全是参考。*
3. **外部参考**：被移出 `SKILL.md` 放入独立文件的参考，通过**上下文指针**访问，仅在指针触发时加载。（涵盖*已披露*参考，例如仍属于 skill 的同级 `GLOSSARY.md` 文件，一直到位于 skill 系统外、任何 skill 都可指向的完全**外部参考**。）

严格的完成标准驱动彻底的**实地工作**，即 agent 在工作中进行的挖掘。无论 skill 是否有步骤都如此，因为“应用每条规则”约束平级参考，正如“完成每个步骤”约束序列。

下推得太少，顶层会膨胀；下推得太多，会隐藏 agent 实际需要的材料。这种张力就是全部决策。

**渐进披露**是沿梯子下移，即从 `SKILL.md` 移到链接文件，使顶层保持可读。机制是 skill 文件夹中以所含内容命名的链接 `.md` 文件（此 skill 将完整定义披露到 `GLOSSARY.md`）。有些 skills 有多种使用方式，每种不同方式是一个**分支**，即不同运行走过 skill 的不同路径。分支是最干净的披露测试：内联每个分支都需要的内容，将仅部分分支会访问的内容置于指针后。**上下文指针**的*措辞*而非目标决定 agent 何时以及多可靠地访问材料。

梯子决定一项内容下放*多远*，**共置**则决定它在该处与*什么并列*：将概念的定义、规则和注意事项保留在同一标题下，而非分散，使阅读一部分时会带上邻近内容。

## 何时拆分

**粒度**是如何细致地划分 skills，每次切分都消耗两种负荷之一，因此仅在切分值得时才拆分。有两种切分：

- **按调用拆分**：当拥有应独立触发的不同**引导词**，或另一 skill 必须访问它时，拆出一个**模型调用** skill。新的始终加载的**描述**需要承担**上下文负荷**，因此独立访问必须值得。
- **按序列拆分**：当仍在前方的**步骤**（某一步的**完成后步骤**）诱使 agent 匆忙完成当前步骤（**过早完成**）时，拆分步骤序列。使它们不在视野中会鼓励 agent 在当前任务上做更多**实地工作**。

## 裁剪

将每个含义保持在**唯一事实来源**中：一个权威位置，使变更行为只需编辑一个位置。

检查每一行的**相关性**：它是否仍与 skill 所做的事有关？

然后逐句而非仅逐行寻找**无操作**：孤立地对每个句子运行无操作测试，一个失败时删除整句而非修剪其中的词。应积极，多数失败的散文应被删除，而非改写。

## 引导词

**引导词**是模型预训练中已有的紧凑概念，agent 在运行 skill 时用它思考，例如*课程*、*战争迷雾*、*垂直切片*。它在全文中重复出现（但不一定，强引导词可能只需一次），通过调用模型已有先验，积累分布式定义，并用最少 token 锚定整片行为区域。

它两次服务于可预测性。在正文中，它锚定*执行*，每次该词出现时 agent 都会采用相同行为。在描述中，它锚定*调用*，当相同词存在于提示词、文档和代码中时，agent 会将这套共享语言链接到 skill 并更可靠地触发它。

寻找将 skills 重构为使用引导词的机会。在三个位置拼写出的三元组（**重复**）、描述用一个句子指向一个想法，这些段落都在要求**折叠**为单个 token。示例包括：

- “快速、确定性、低开销” → *紧凑*，将贯穿一个阶段反复陈述的质量转化为单个预训练词（一个*紧凑*循环）。
- “你相信的循环” → *红*，将模糊关卡转化为二元可观察状态（循环在缺陷上变*红*，或不变红）。

你会两次获益：更少 token，*以及*更尖锐的钩子供 agent 挂靠思考。假定每个 skill 都携带可以由引导词淘汰的重复陈述，去找出它们。

## 失败模式

使用这些诊断用户在使用 skill 时可能遇到的问题。

- **过早完成**：在真正完成前结束一步，注意力滑向*完成这件事*。按顺序防御：先明确完成标准（低成本、局部）；仅当它不可避免地模糊且观察到匆忙时，才通过拆分隐藏完成后步骤（序列切分）。
- **重复**：相同含义出现在多个位置。消耗维护成本和 token，并将一个含义在梯子上的显著性夸大到超过其真实等级。
- **沉积物**：因为添加感觉安全、删除感觉有风险而沉积的陈旧层。任何没有裁剪纪律的 skill 的默认命运。
- **蔓延**：skill 单纯太长，即使每一行都是活跃且唯一的。伤害可读性和可维护性，并浪费 token。解法是梯子：在指针后披露**参考**，并按**分支**或序列拆分，使每条路径只携带所需内容。
- **无操作**：模型默认已遵循的一行，因此你付出负荷却没有表达内容。测试是：与默认行为相比它是否改变行为？弱引导词（agent 已经大致充分时使用*要彻底*）是无操作；修复方法是更强的词（*持续追问*），而非不同技术。
- **否定**：通过禁止引导会适得其反，*不要想象一头大象*命名大象并使它更可得，而非更不可得。提示**正向**内容，说明目标行为，使被禁止行为从未被说出；仅将禁止保留为无法正向表达的硬护栏，即使这样，也要与替代做法配对。
