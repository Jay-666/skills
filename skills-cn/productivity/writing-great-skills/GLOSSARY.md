# 术语表：构建优秀 Skills

这是定义优秀 skill 的领域模型。skill 的存在是为了从随机系统中驾驭确定性；根本美德是**可预测性**，下方每个术语都是作用于它的杠杆。这是 [`writing-great-skills`](SKILL.md) 的已披露参考。

术语按维度分组：**调用**（如何访问 skill）、**信息层级**（如何组织其内容）、**引导**（如何塑造 agent 的运行时行为）和**裁剪**（如何保持精简）。每种**失败模式**都位于能治愈它的杠杆旁，并标记为*失败模式*。

任意定义中的**加粗术语**也在此术语表中定义，可通过其标题找到。

## 可预测性

skill 使 agent 每次运行都以相同*方式*行动的程度，即相同过程而非相同输出（头脑风暴 skill 应可预测地发散，其 token 可变，行为不变）。这是所有其他术语服务的根本美德，成本和可维护性是其症状，而非竞争项。

_避免使用_：一致性、可靠性、健壮性、输出确定性

## 调用

如何访问 skill，以及为选择付出的两种负荷。

### 模型调用

保留其 **description** 字段的 skill，因此 agent 能看见并自主触发它，人仍可输入其名称，所以模型调用总是*包含*用户访问。不存在仅模型状态：描述仅会*增加* agent 发现能力，绝不会移除人的访问能力。它以每轮永久的**上下文负荷**交换这种可发现性。其他 skills 可访问它，因为让 agent 可发现的描述也使其可调用。内容全为**参考**的模型调用 skill 也是共享参考的一种归宿：其他 skill 可调用它，因此多个 skills 所需的参考位于一个位置。仅当 agent 必须自行访问该 skill 时选择模型调用；若它只会人工触发，移除描述且不承担上下文负荷。

_避免使用_：能力、工具、功能

### 用户调用

描述被移除的 skill，对 agent 不可见，只能由人输入其名称访问（仅用户，而**模型调用**是用户*和 agent*）。用 agent 可发现性交换零**上下文负荷**。因为没有描述，除人以外没有任何对象可访问它，其他 skill 无法触发它。

_避免使用_：过程、工作流、命令

### 描述

skill 的机器可读触发条件，也是**模型调用** skill 被迫始终保持加载的一个**上下文指针**。它的存在本身*就是*调用维度：保留它，skill 即为模型调用（并可由其他 skills 访问）；删除它，skill 即为**用户调用**，仅人可访问。它是模型调用 skill **上下文负荷**的来源。

_避免使用_：frontmatter、摘要

### 上下文指针

保存在 agent 上下文中的引用，命名某些上下文外材料并编码访问它的条件。**描述**是顶层上下文指针（上下文窗口 → skill）；指向已披露文件的指针是下一级的同类对象。其措辞而非目标决定 agent *何时*访问，以及*多可靠地*访问。位于措辞薄弱指针后的必需目标是方差缺陷：先修复措辞，仅当明确化失败时才内联材料。

_避免使用_：链接、引用、导入

### 上下文负荷

**模型调用** skill 施加在 agent 上下文窗口上的成本，即其始终加载、同时消耗 token 和注意力的**描述**。这是**用户调用** skills 因没有描述而逃避的成本，也是拆分出更多模型调用 skills 的制动。

_避免使用_：token 成本、上下文膨胀

### 认知负荷

**用户调用** skill 施加在人身上的成本，即必须记在脑中的内容：有哪些 skills，以及何时使用各自（人是索引）。这是**模型调用**通过 agent 可发现性移除的成本，也是拆分出更多用户调用 skills 的制动。它不是要最小化的成本，而是人类主导权的代价，也是某些 skills 保持用户调用的原因。在人的判断重要处承担它，在不重要处移除它。

_避免使用_：人工索引、负担、开销

### 路由 Skill

一种**用户调用** skill，其职责是指向其他用户调用 skills，命名每一个以及何时使用它，使人只需记住一个 skill 而非多个。它只能提示，绝不能触发它们：用户调用 skills 没有**描述**，因此除人以外没有任何对象可访问它们。当用户调用 skills 增多时，它是**认知负荷**的解药。

_避免使用_：分发器、菜单、注册表、索引、路由过程

### 粒度

如何细致划分 skills。更细的划分消耗两种负荷之一：更多**模型调用** skills 消耗**上下文负荷**（更多描述拥挤窗口并竞争注意力）；更多**用户调用** skills 消耗**认知负荷**（人需要记住和使用的更多）。两种切分引导划分。按**调用**切分时，在拥有可触发的不同**引导词**处拆出模型调用 skill，即实际在提示词中使用的触发词。按**序列**切分时，在某一步的**完成后步骤**需要隐藏处拆分一串**步骤**，因为将其隔离到自身上下文会清除后续内容。注意反向操作：合并序列会向前面步骤暴露其后的完成后步骤，引发过早完成。

_避免使用_：分块、模块化

## 信息层级

如何安排 skill 内容，以及每一项位于梯子下方多远。

### 信息层级

按 agent 需要内容的即时程度对 skill 内容排序的单一梯子，由两种切分产生：在文件内或在指针后，以及步骤或参考。层级如下：

- **步骤**：文件内，主要
- **参考**：文件内，次要
- **参考**：已披露，位于**上下文指针**后

没有**步骤**的 skill 仅使用底部两层，通常是合理的平级集合（例如审查的每条规则在同一层），这是一种良好安排而非异味。层级独立于调用方式：无论全是步骤、全是参考还是两者兼有，skill 都可由模型或用户调用。当 skill 有步骤时，本应披露的文件内参考会埋没它们，并使注意到它们成为抛硬币，这不仅影响可读性，还是方差杠杆。保持梯子顶端可读，尽可能将内容下推。

_避免使用_：结构、组织、布局

### 步骤

agent 执行的有序动作。当 skill 有步骤时，它们是内容的主要层级，也是使其在 SKILL.md 中占据位置的部分。并非每个 skill 都有步骤：skill 可全是步骤（`tdd`）、全是**参考**（审查），或两者兼有，与调用方式无关。每一步以清晰或模糊的**完成标准**结束。

_避免使用_：工作流、指令、编排

### 参考

agent 按需引用的材料，例如定义、事实、参数、示例和条件指令。当 skill 有**步骤**时，它次于步骤；当 skill 没有步骤时，它就是全部内容；或者它完全位于任何 skill 外，参见**外部参考**。通过**上下文指针**访问，是**渐进披露**的首要候选项。

_避免使用_：辅助材料、文档、背景

### 外部参考

位于 skill 系统外的**参考**，即普通文件，没有**描述**、没有**步骤**、不可调用，任何 skill 都可指向它。它是无需自行触发的共享参考的归宿，也是两个**用户调用** skills 唯一可用的共享归宿，因为两者均没有描述，因此都不能触发对方。

_避免使用_：文档、资源、知识库

### 渐进披露

将**参考**沿梯子下移，即移出 SKILL.md 并置于**上下文指针**后，使顶端保持可读。这主要不是 token 优化，而是保护**信息层级**的方式。它由**分支**授权：披露仅部分分支需要的内容，内联每条路径均需要的内容；若指针在必需材料上触发不可靠，则明确其措辞，仅在失败时将材料拉回内联。

_避免使用_：延迟加载、分块

### 共置

将 agent 同时需要的材料放在一个位置，即将概念的定义、规则和注意事项放在一个标题下，而非分散在文件中，使阅读一部分时带上邻近内容。它是文件内**信息层级**的伙伴：层级排序一项内容下放*多远*，共置决定它到达后与*什么并列*。**参考**正文的正确格式没有公式，测试是 skill 应读起来像为 agent 编写的文档，分组材料在此方面优于分散材料。它不同于**重复**：重复是在两处重复一个含义，分散则将单个含义碎片化到多处。

_避免使用_：分组、聚类、内聚性

### 蔓延

*失败模式。* 单纯过长的 skill，即 SKILL.md 中行数过多，与它们是否陈旧或重复无关。即使所有内容都活跃且唯一，skill 仍可蔓延。它消耗可读性（agent 在能行动前必须穿过更多内容，注意力在过量内容中变薄）、可维护性（每一额外行都要保持**相关**）和 token。解法是**信息层级**：将**参考**推到**上下文指针**后，并按**分支**或序列拆分，使每条路径只携带所需内容。它不同于**沉积物**（陈旧积累导致的长度）和**重复**（重复含义导致的长度），蔓延就是长度本身，无论其原因。

_避免使用_：膨胀、长度、大小、冗长

## 引导

将 agent 运行时行为塑造成**可预测性**的杠杆。

### 分支

skill 可被调用的一种不同方式，即 skill 处理的一种情形，使不同运行走过不同路径。拥有许多步骤的 skill 可能携带许多分支，线性 skill 没有分支。

_避免使用_：路径、情形、分叉

### 引导词

模型预训练中已有的紧凑概念，也称 _Leitwort_，agent 在运行 skill 时用它思考。它通过调用模型已有先验，以最少 token 编码行为原则，例如*课程*、*最近发展区*、*战争迷雾*、*垂直切片*。它作为 token 重复出现，而非作为句子，在 skill 中积累分布式定义并锚定整片行为区域。若清晰定义，自创词也可使用，但自创词无法调用先验，你需用定义 token 支付预训练词免费提供的内容。优先使用已有词。

引导词两次服务于**可预测性**。在正文中，它锚定**执行**，每次概念出现时 agent 都采用相同行为；在平级参考中，它聚焦对一类事物的注意力，并在每次运行中调用正确检查。在**描述**中，它锚定**调用**，且不只在 skill 内：当相同词存在于提示词、文档和代码库中时，agent 将这套共享语言链接到 skill 并更可靠地触发它。描述应使用你想要该 skill 时实际使用的引导词。

_避免使用_：关键词、术语、主题

### 完成标准

告知 agent 一个工作单元完成的条件，即它判断时对照的目标。两项属性使它成为杠杆而非仅是质量。其**清晰度**（agent 能区分完成和未完成吗？）抵抗**过早完成**，模糊边界（“已达成理解”）会让 agent 宣告完成并滑向下一步；此维度需要*步骤*才能生效，因为过早完成是步骤间失败。其**要求度**（要求多少）设定**实地工作**，“已覆盖每个修改模型”会强制彻底工作，而“生成变更列表”不会；此维度*不*受步骤限制，也可约束平级参考正文，这就是没有步骤的 skill 仍可带有穷尽性门槛（“应用每条规则”）的方式。最强标准既可检查又穷尽。

_避免使用_：完成条件、退出条件、停止规则

### 实地工作

agent 在单一步骤中幕后完成的工作，例如读文件、探索代码库、做出变更、挖掘所需内容而非转交给用户。它位于步骤结构之下，绝不写为自身步骤，潜在于措辞中，由 agent 而非 skill 控制。它是**完成后步骤**跨步骤拉力在步骤内的对应物。由**引导词**（*全面*、*彻底*）或要求工作穷尽的**完成标准**提高，包括应用于平级参考的要求度维度，这推动平级参考 skill 覆盖其所有层级。缺少该要求或**过早完成**提前截断步骤时，它会变薄。

_避免使用_：范围、努力、勤勉、覆盖率

### 完成后步骤

当前步骤之后的**步骤**。它们可见时，会将 agent 向前拉入**过早完成**，看到越多，拉力越强；防御方式是将步骤序列拆为两部分来隐藏它们。

_避免使用_：地平线、战争迷雾、前瞻

### 过早完成

*失败模式。* 在当前步骤真正完成前结束它，因为 agent 的注意力滑向完成这件事而非工作本身。这是步骤间失败，发生需要**步骤**：没有步骤的 skill 提前退出不是过早完成，而是未满足要求下薄弱的**实地工作**。两种力量的拉锯：可见的**完成后步骤**（向前拉力）和**完成标准**的清晰度（阻力，清晰可检查的门槛能保持，模糊门槛会让步）。模糊性是必要条件：无论有多少后续步骤可见，清晰边界都会抵抗拉力，因此从不匆忙的步骤无需防御。两种杠杆能维持会匆忙的步骤，但应按顺序使用：**先明确边界**，它局部且廉价。仅当标准不可避免地模糊*且*确实观察到匆忙时，才**隐藏后续步骤**，且隐藏仅能跨真实上下文边界生效（用户调用交接或子 agent 调度；内联模型调用会将后续步骤留在上下文中，无法清除任何内容）。它是实地工作变薄的一个原因，但不同于它：即使步骤运行到完全完成，实地工作仍可能薄弱。

_避免使用_：过早关闭、匆忙、赶工、走捷径

### 否定

*失败模式。* 通过禁止进行引导，即告诉 agent *不要*做什么，会将被禁止行为拖入上下文，使其更容易获得而非更难。*不要想象一头大象*，大象便成了唯一内容；*绝不写冗长注释*，冗长就成了 agent 刚读到的模式。否定是一个弱修饰词，被强激活概念压过，因此禁令有一半被读作执行该行为的指令。其**引导词**是*大象*，即禁止命名进入画框的任何事物。解法是提示**正向**内容，描述目标行为（“编写一行注释”），使被禁止内容从未被说出。禁止仅应作为无法正向表述行为上的硬护栏占据位置；即使如此，也要与正向目标配对，使注意力落在应做什么。

_避免使用_：讽刺性反弹、不要提示、粉红大象

## 裁剪

保持 skill 精简，每种补救措施与其治愈的失败配对。

### 唯一事实来源

每个含义恰好位于一个权威位置的理想状态，因此 skill 行为变更只需在一个位置修改。**重复**是对它的违反。

_避免使用_：归宿、规范位置

### 重复

*失败模式。* 相同含义被赋予多个**唯一事实来源**。它消耗维护成本（改变一个位置，必须改变其他位置）、消耗 token，并夸大显著性，重复一个含义会使它在梯子上的权重超过真实等级。它是**引导词**的意外反面，引导词通过有意重复 token 而非含义提高注意力。

_避免使用_：复述、冗余

### 相关性

一行是否仍与 skill 所做的事相关，是决定保留什么的镜头。一行会因从未与任务相关（纯粹说明，或应被披露的**分支**）或因变得陈旧而失去相关性，即其描述的行为或世界变化时逐渐过时。更短的 skills 更容易保持相关，因为检查每一行的成本更低。它不同于**无操作**：相关性问一行是否与任务相关，而非是否改变行为。

_避免使用_：承重、陈旧性、新鲜度

### 沉积物

*失败模式。* 因为添加感觉安全、删除感觉有风险而沉积在 skill 中且从未清除的旧内容层，使陈旧且无关的行积累，你必须钻穿它们才能找到仍活跃的内容。它是没有裁剪纪律的任何 skill 的默认命运，是对**相关性**的缓慢侵蚀，区别于**重复**中的重复含义。

_避免使用_：堆积、膨胀、冗余废料、腐朽

### 无操作

*失败模式。* 不改变任何内容的指令，因为模型默认已会这样做，你付出负荷来告诉 agent 它无论如何都会做的事。测试是：与默认行为相比，该行是否改变行为？一行可完全**相关**但仍是无操作。使**引导词**免费的相同先验也使无操作毫无价值。

引导词是一种*技术*，无操作是对一行的*判决*，二者交叉。弱到无法超过默认行为的引导词是无操作（agent 已经大致充分时使用*要彻底*），修复方式是使用通过判决的更强词（*持续追问*），而非不同技术。因此无操作测试，即它是否相对默认改变行为，也是评判引导词是否配得上其重复的方式。这是相对模型而非相对读者的：两个人对一行是否为无操作的分歧，是对默认行为的分歧，应通过运行 skill 而非辩论来解决。

_避免使用_：冗余指令、重述显而易见内容、过度阐述
