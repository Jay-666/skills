一个松散想法到来：它大到超出单个 agent 会话，并被迷雾包裹，从这里到**目标**的路径尚不可见。路径规划旨在找到这条路径，而非直接冲向目标。此 skill 在仓库的 issue 跟踪器上将路径绘制为一张**共享地图**，然后逐项处理其**决策任务单**，即解答后形成决策的问题，而非要执行的构建切片，直至路线清晰。

目标因工作而异，命名它是绘图的第一个动作，它塑造每一张任务单。它可能是一份需要交接并迭代的规格说明、规划开始前需要锁定的决策，或就地完成的变更，例如数据结构迁移。地图与领域无关，工程工作、课程内容，任何符合该形态的内容都可以使用。

## 规划，而非执行

路径规划默认是**规划**：每张任务单解决一个决策，路径清晰时地图即完成，即在有人去做这件事前不再有待决事项。想要直接执行工作的冲动通常是已到达地图边缘、该交接的信号。一项工作可在其**备注**中覆盖此规则，将执行带入地图本身，但若无此说明，应产出决策而非交付物。

## 通过名称引用

每张地图和任务单都是 issue，因此都有**名称**，即标题。在人阅读的所有内容中，包括叙述和地图的“迄今决策”，均通过名称引用它，绝不可只用 id、编号或 slug。一排 `#42, #43, #44` 不可读，名称则一眼可读。id 和 URL 不会消失，名称会包装其链接，但它们置于名称*内部*，绝不替代名称。

## 地图

地图是此仓库 issue 跟踪器中的一个单独 issue，带有 `wayfinder:map` 标签，是规范工件。它的任务单是地图的子 issue。

地图是**索引**，不是存储库。它列出已作出的决策，并指向保存细节的任务单；一个决策只存在于一个位置，即其任务单，因此地图绝不重复陈述，只给出摘要和链接。

**地图、子任务单、阻塞关系和前沿查询实际位于何处取决于跟踪器。** 应已提供 issue 跟踪器；若未提供，运行 `/setup-matt-pocock-skills`。查阅跟踪器文档的“路径规划操作”章节，了解*此*仓库如何表达它们。若未提供跟踪器，默认使用本地 Markdown 跟踪器。

### 地图正文

整张地图的低分辨率视图，每个会话只加载一次。**不**列出开放任务单，它们是通过查询找到的开放子 issue。

```markdown
## 目标

<到达此地图终点时的状态，即此工作正寻找路径抵达的规格、决策或变更。用一两行表达；每个会话在选择任务单前都要以此为方向。>

## 备注

<领域；每个会话应查阅的 skills；本工作持续适用的偏好>
```

### 任务单

每张任务单都是地图的**子 issue**，跟踪器的 issue id 是其身份。其正文是问题，规模适合一个 100K token 的 agent 会话：

```markdown
## 问题

<此任务单解决的决策或调查>
```

每张任务单带有 `wayfinder:<type>` 标签，即 `research`、`prototype`、`grilling`、`task` 之一（参见[任务单类型](#任务单类型)）。

会话在任何工作前，**首先**将任务单分配给驱动地图的开发者，以此**认领**任务单，从而使并发会话跳过它。该被分配者*就是*认领：开放且未分配的任务单即未被认领。

阻塞使用跟踪器的**原生**依赖关系，这一点至关重要，因为它在跟踪器自身 UI 中以*可视化*方式渲染前沿，人无需打开地图就能看到哪些内容可获取。仅缺少原生阻塞的跟踪器才回退到正文约定。当阻塞一张任务单的所有任务单都关闭时，它即为**未阻塞**；**前沿**是开放、未阻塞、未认领的子任务单，即已知的边缘。

答案不是正文的一部分，而是在解决时记录（参见[处理地图](#处理地图)）。解决任务单时创建的资产应从 issue 链接，而非粘贴进去。

## 任务单类型

每张任务单要么是 **HITL**，即人参与循环，与能代表自己发言的人*共同*处理；要么是 **AFK**，仅由 agent 驱动。HITL 任务单只能通过这种实时交流解决，agent 绝不可代替人的一方回答（一个自问自答的深入追问 agent 已破坏此规则）。

- **研究**（AFK）：阅读文档、第三方 API 或知识库等本地资源，以揭示某项决策等待的事实。由 `/research` **子 agent** 解决。当前工作目录之外的知识是必需条件时使用。
- **原型**（HITL）：制作廉价、粗糙但具体的工件供讨论反应，例如大纲、粗略方案、stub，或通过 /prototype skill 创建 UI 或逻辑代码，以提高讨论保真度。将原型作为资产链接。当“它应是什么样子”或“它应如何行为”是关键问题时使用。
- **深入追问**（HITL）：通过 /grilling 和 /domain-modeling skills 进行对话，一次一个问题。默认情形。
- **任务**（HITL 或 AFK）：在能作出*决策*前必须完成的手动工作，没有内容需要决策、原型化或研究，但讨论在完成前受阻。为判断某服务 API 而注册服务、配置访问权限、移动数据以查看其形态。这是唯一会*执行*而非决策的类型，它之所以有资格存在是因为解除决策阻塞，而非交付目标。agent 在可行时独自驱动（AFK），否则交给人一个精确检查清单（HITL）。工作完成时解决，答案记录已完成内容及后续任务单所依赖的事实（凭证位置、新 URL、行数）。

## 战争迷雾

地图有意保持*不完整*，不要绘制尚不可见的内容。开放任务单之外是**战争迷雾**，即你知道即将出现、但尚不能确定的决策和调查的模糊图景，因为它们依赖仍未解决的问题。解决一张任务单会清除其前方迷雾，将现可明确的内容逐项毕业为新任务单，直至通往目标的路径清晰且不再有任务单。

地图的**尚未明确**章节记录这种模糊图景：推测的问题、之后要重新访问的区域。这是*通往*目标的未发现前沿，所有内容都在范围内，只是不够明确到能创建任务单。按当前视野允许的松散或完整程度记录；它也为协作者指示此工作前往何方。

**迷雾还是任务单？** 判断标准是你现在能否精确陈述问题，而非现在能否回答它。

- 当问题已经明确时创建**任务单**，即使它被阻塞且暂时无法行动。
- 当还无法如此清晰地表达问题时记为**尚未明确**。不要预先将迷雾切成任务单大小的片段，它比任务单更粗糙；当前沿到达时，一个补丁可能毕业成多张任务单，也可能一张也没有。

**尚未明确**排除已经决定的内容（迄今决策）、已经是开放任务单的内容和范围外内容（下一节）。

## 范围外

迷雾只会朝*目标*聚集。目标固定范围，因此超出目标的工作属于**范围外**，它不是迷雾，也不属于**尚未明确**。它在地图上有自己的**范围外**章节，记录你有意排除在*本次*工作之外的内容。使其落在这里的是范围，而非明确程度。

范围外工作永不毕业，前沿在目标处停止，因此仅当重新绘制目标时才会返回，而且将作为新的工作而非恢复旧工作。

判定内容范围外是范围界定动作，而非路线上的一步。当已存在任务单被发现位于目标之后，例如绘图时错误纳入范围，或由一次解决暴露，应将其**关闭**（关闭任务单明确不在前沿），并在**范围外**章节留下一行：摘要、为何范围外以及指向已关闭任务单的链接。它应保留在**迄今决策**之外，该章节记录实际走过的路线，范围边界不是路线的一步。

## 调用

两种模式。无论哪种，**每个会话绝不解决超过一张任务单**，研究任务单除外。

### 绘制地图

用户使用一个松散想法调用。

1. **命名目标。** 运行一次 `/grilling` 和 `/domain-modeling` 会话，确定此地图要找路抵达的内容，即规格、决策或变更。目标固定范围，因此先解决它。
2. **映射前沿。** 再次深入追问，此次采用**广度优先**：横向展开整个空间，而非在某一线程上深入，揭示开放决策和当前可执行的第一步。**若未出现迷雾**，即通往目标的路径已经清晰且全程足以在一个会话内完成，则不需要地图。停止并询问用户希望如何继续。
3. **创建地图**（标签 `wayfinder:map`）：填写目标和备注，迄今决策为空，在**尚未明确**中勾勒迷雾。
4. 将**现在能明确的任务单**创建为地图的子 issue，然后在**第二轮**接入阻塞边（issues 需要 id 才能互相引用）。接入将它们分为前沿和被阻塞项；所有尚不能明确的内容留在迷雾中，即**尚未明确**章节。
5. **启动研究子 agent。** 对每张刚创建的 `research` 任务单，启动一个 `/research` 子 agent 并行解决，并将发现捕获在一次性的 `research/<name>` 分支中，任务单提供指向该分支的上下文指针。
6. 停止，绘图是一个会话的工作，它不会亲自解决任何任务单。

### 处理地图

用户使用一张地图（URL 或编号）调用。任务单是**可选**的，若未指定，由你而非用户选择下一个决策。

1. 加载**地图**，即低分辨率视图，而非每张任务单正文。
2. 选择任务单。若用户指定一张则使用它，否则按顺序取第一张前沿任务单。**认领它**：在任何工作前将其分配给自己。
3. 解决它，**按需放大**：按需获取任一相关或已关闭任务单的完整正文；调用 `## 备注` 块中命名的 skills。若不确定，使用 `/grilling` 和 `/domain-modeling`。
4. 记录解决结果：以**解决评论**发布答案，**关闭** issue，并向地图的“迄今决策”**追加上下文指针**。
5. 添加新浮现的任务单（先创建后接入）；毕业答案已使其可明确的所有迷雾，从**尚未明确**中移除每个已毕业块，使其仅存在于新任务单中。若答案揭示某张任务单，无论当前或其他任务单，位于目标之外，应将其**判定为范围外**，而非在路线中解决它。若决策使地图其他部分失效，更新或删除这些任务单。

用户可能并行运行未阻塞任务单，因此应预期其他会话会并发编辑跟踪器。
